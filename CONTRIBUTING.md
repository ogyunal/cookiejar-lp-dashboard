## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/yourusername/cookiejar-landing.git`
3. Create a branch: `git checkout -b feature/your-feature-name`
4. Make your changes
5. Test thoroughly
6. Commit: `git commit -m "Add your feature"`
7. Push: `git push origin feature/your-feature-name`
8. Open a Pull Request

## Code Style

- Use JavaScript (not TypeScript)
- Follow the existing code structure
- Use functional components
- Use Tailwind CSS for styling
- Keep components small and focused

## Commit Messages

Format: `[Type] Description`

Types:
- `[Feature]` - New feature
- `[Fix]` - Bug fix
- `[Style]` - UI/UX improvements
- `[Refactor]` - Code refactoring
- `[Docs]` - Documentation updates
- `[Test]` - Test additions/updates

Examples:
- `[Feature] Add dark mode toggle`
- `[Fix] Resolve file upload timeout`
- `[Style] Improve mobile navigation`

## Pull Request Process

1. Update README.md if needed
2. Test on both desktop and mobile
3. Ensure no console errors
4. Verify authentication still works
5. Check file uploads work
6. Test in multiple browsers

## Questions?

Open an issue or reach out to the maintainers!
